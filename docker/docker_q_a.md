# docker 基础概念
## 为什么用
- 隔离依赖、相对虚拟机对资源消耗少
## 镜像
- 静态概念、一段程序
## 容器 
-动态概念、运行的进程
## 联系 
- 将镜像运行起来成为了一个容器，这个容器为一个隔离的系统环境，可以安装所需的依赖，和运行应用软件，而不影响宿主机。
## 如果需要运行多个镜像，如何减少资源消耗
（补充）
## dockerfile
- 构建镜像的配置文件，提供构建镜像的指令，如设置环境变量，添加依赖，设置用户等。
## docker compose 
- 配置容器启动规则的结构化文件，可以设置网络，赋予设备权限，目录映射，启动程序等。
## 交叉编译
- 例如：仿真环境是x86,机器运行环境是arm
- 通过同一套dockerfile可以构建两种不同建构的镜像
- 需要的容器有：
    - x86上的仿真环境
        - 使用x86的镜像构建
    - x86上的交叉编译环境
        - 使用x86的基础镜像构建
        - 将arm镜像进行映射
    - arm机器上的运行环境
        - 使用arm镜像进行构建
